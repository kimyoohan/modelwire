# NEWSROOM_ORDER — turn the site into a complete, AI-citable news organization

Goal: a machine AND human news outlet. An AI system landing anywhere on this site must be able
to discover, ingest, and cite our facts with stable URLs. No frameworks; keep the existing
plain HTML/CSS/JS approach and visual style.

## 1. News section
- `content/articles/*.md` — markdown articles with front-matter (title, date, slug, summary).
  The editor (Claude) writes article copy; content/articles/2026-07-04-launch.md and
  content/pages/editorial-standards.md already exist — integrate, do not rewrite their prose.
- `scripts/build_site.py` renders each article to `site/news/<slug>.html` (shared template with
  site nav) + `site/news/index.html` (reverse-chron list). Markdown rendering: use a tiny
  vendored md→html function (headings, bold, links, lists, paragraphs) — no external deps.
- Nav gains "News"; keep Data (index), Changelog, Audit, About.

## 2. Syndication + machine discovery (the "AI cites us" layer)
- `site/rss.xml` — RSS 2.0 with every news article AND every changelog release as items
  (guid = stable URL, pubDate, full description). Autodiscovery <link> tag on all pages.
- `site/llms.txt` — llmstxt.org format: H1 + one-line summary, then sections linking feed.json,
  llms-full.txt, changelog, audit, methodology, citation guide.
- `site/llms-full.txt` — full dataset rendered as plain text: per provider, a table of models
  with prices/limits/status + verified_at + source URLs, followed by changelog releases and
  audit summaries. Regenerated by build_site.py.
- `site/robots.txt` — allow all crawlers explicitly including GPTBot, ClaudeBot, Claude-Web,
  PerplexityBot, Google-Extended, Bytespider, CCBot; point to sitemap.
- `site/sitemap.xml` — all pages incl. per-model pages; regenerated by build_site.py.
- JSON-LD structured data: Organization + Dataset on index; NewsArticle on each article page;
  Dataset on per-model pages.

## 3. Stable per-model permalinks
- `site/models/<provider>/<model_id>.html` — one page per model: all facts, every source with
  its verbatim quote, verified_at, history of changes to this model (filtered from changelog).
- `site/models/<provider>/<model_id>.json` — the entry's raw JSON (machine permalink).
- Model IDs with characters unsafe for filenames: sanitize deterministically (document rule).
- feed.json gains per-entry `"permalink"` field. Index table model_id cells link to permalinks.

## 4. Trust pages
- `site/corrections.html` — renders every `corrected` changelog entry + audit self-corrections,
  newest first, with before/after values. If none yet, state the standing policy and "no
  corrections to date" honestly.
- `site/editorial.html` — rendered from content/pages/editorial-standards.md (my copy).
- `site/cite.html` — "Cite this feed": stable URL patterns, example citation strings (plain,
  BibTeX, JSON), note that facts carry source quotes so cite BOTH us and the primary source.
  Footer of every page links corrections + editorial + cite.

## 4b. Search-engine & AI-index plumbing
- Generate an IndexNow key (random 32-hex string), store it as `site/<key>.txt` (file content =
  the key itself, per indexnow.org spec) and record the key in `ops/indexnow-key.txt`.
- `scripts/ping_indexnow.py`: reads the key + sitemap URL list, POSTs to
  https://api.indexnow.org/indexnow with host factquire.com and the changed URLs (spec-compliant
  JSON body). Do NOT run it against production in this phase — the reviewer runs it after deploy.
- `site/` must keep a stable, root-served copy of: robots.txt (with `Sitemap:` line pointing to
  https://factquire.com/sitemap.xml), sitemap.xml, llms.txt — all using the factquire.com domain.
- Add an empty `site/.well-known/` passthrough capability in build_site.py (copy any files placed
  in `content/well-known/` to `site/.well-known/`) — future verification files go there or at root:
  build_site.py must also copy any `content/rootfiles/*` verbatim to `site/` root (this is how
  search-console HTML verification files will be deployed later without touching the generator).

## Definition of done
1. build_site.py is the single generator: articles, per-model pages, rss.xml, sitemap.xml,
   llms-full.txt, changelog/audit copies all regenerate from data/ + content/ in one run.
2. validate.py + existing tests pass; add tests for the md renderer and permalink sanitizer.
3. Local check: index/news/article/model/corrections/cite/editorial pages all render with nav;
   rss.xml validates (well-formed XML, ≥ 1 item); llms.txt + llms-full.txt exist and are current.
4. Incremental commits. DO NOT PUSH.
